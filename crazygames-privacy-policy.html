中文

当前 CrazyGames 版本会在浏览器本地保存游戏进度、金币、免费提示次数、教程与挑战引导状态，以及音乐、音效、语言等设备偏好设置，用于保证游戏正常运行并恢复进度。

当某个 CrazyGames 候选版本启用 telemetry 时，游戏会向项目维护者自建的 first-party telemetry 服务发送匿名或去标识化的运行与交互事件，用于产品运营、关卡分析和版本验证。

当前 telemetry 默认使用 session_only 身份模式，不会默认在浏览器本地保留跨会话 telemetry player_id。发送的页面地址字段会裁剪为 origin + pathname，不会发送 query string 或 hash。

当前 Basic Launch 范围不包含外部广告 SDK、外部登录选项，也不要求 CrazyGames 账号登录后才能游玩。

如果后续版本新增 rewarded ads、登录、支付或其他第三方 SDK，本页面会同步更新。

English

The current CrazyGames build stores gameplay progress, coins, free hints, tutorial and challenge guidance states, and device preferences such as music, SFX, and language locally in the browser so the game can function and restore progress.

When CrazyGames telemetry is enabled for a candidate, this build sends anonymous or pseudonymous runtime and interaction events to a first-party telemetry service operated by the project maintainer for product operations, level analysis, and release verification.

The current telemetry posture uses session_only identity and does not keep a cross-session telemetry player_id in browser storage by default. Browser address fields are trimmed to origin + pathname; query strings and hashes are not sent.

The current Basic Launch scope does not include external ad SDKs or external login options, and it does not require CrazyGames account sign-in before play.

If later versions add rewarded ads, sign-in, payments, or other third-party SDKs, this page will be updated together.
