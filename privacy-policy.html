Paws To Pass / 鼠你能过

Updated: 2026-05-14 · Public scope: itch-web HTML5 browser build

中文

当前 itch-web 公开版会在浏览器本地处理关卡进度、金币、免费提示次数、奖励记录以及音乐 / 音效 / 语言等设置。这些数据主要用于让游戏正常运行和恢复进度。

当前版本会向项目维护方自建的 telemetry 服务发送匿名/去标识化运行与交互事件，用于统计会话、关卡、退出、提示、菜单、设置和浏览器运行情况，并生成按天、按关卡的聚合报表。

当前事件可能包含会话标识、会话内匿名玩家标识、事件时间、关卡编号、结果、退出去向、语言、视口尺寸、设备类型、当前页面的 origin + pathname、referrer 来源 origin、存储模式和会话级匿名身份模式。

当前公开版不包含奖励广告、应用内购买、账号登录、云存档、玩家聊天或基于真实身份的画像系统。

当前公开版会主动裁剪浏览器地址字段：pageUrl 只保留 origin + pathname，referrer 只保留来源 origin，不发送 query string、hash 或临时分享参数。

当前公开版 itch-web telemetry 现按 session_only 模式运行，会在当前浏览器会话内使用匿名 player_id 关联事件，但不再默认在浏览器本地保留跨会话 telemetry player_id。

原始事件当前按短期保留策略处理，Cloudflare Worker 环境默认将 raw_events 保留 30 天；按天、按关卡聚合后的统计可保留更久，用于运营分析、发版复盘和趋势比较。

作为底层托管、传输和存储基础设施，Cloudflare 还可能基于服务安全、网络传输与日志处理使用必要服务性 cookies 或网络层标识。

如后续版本新增 Billing、登录、广告或其他第三方 SDK，本页面会同步更新。

English

The current public itch-web build stores level progress, coins, free hints, reward records, and settings such as music, SFX, and language locally in the browser so the game can function and restore progress.

The current build sends anonymous or pseudonymous runtime and interaction events to a first-party telemetry service for session, level, exit, hint, menu, settings, and browser-runtime reporting, including daily aggregated gameplay metrics.

The current event stream may include session identifiers, session-scoped anonymous player identifiers, event time, level id, result, exit-target fields, language, viewport size, device class, the current page origin + pathname, referrer origin, storage mode, and session-level identity mode.

The current public build does not include rewarded ads, in-app purchases, account sign-in, cloud save, player chat, or real-identity profiling.

The current public build trims browser address fields before sending telemetry: pageUrl keeps only origin + pathname, and referrer keeps only the source origin. Query strings, hashes, and temporary share parameters are not sent.

The current public itch-web telemetry configuration now runs in session_only mode. It uses an anonymous player_id within the active browser session to correlate events, but it does not keep a cross-session telemetry player_id in local browser storage by default.

Raw events are currently kept on a short retention window. The Cloudflare Worker configuration keeps raw_events for 30 days by default, while aggregated per-day gameplay metrics may be retained longer for release review and product analytics.

As the hosting, transport, and storage infrastructure provider, Cloudflare may also use necessary service cookies or network-layer identifiers for service security, delivery, and logging.

If later versions add billing, sign-in, ads, or other SDKs, this page will be updated together with the runtime notices.
